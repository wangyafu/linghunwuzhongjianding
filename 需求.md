

# 产品需求文档 (PRD)：灵魂物种档案馆 (Soul Species Archive)

| 文档版本 | V1.0 |
| --- | --- |
| **产品类型** | GenAI 泛娱乐/轻应用 (H5 / 小程序) |
| **核心场景** | 小红书/朋友圈分享、情绪宣泄、身份认同 |
| **设计风格** | 复古百科全书 / 博物学档案馆风格 |
| **核心策略** | 低成本文案生成 + 预制图片库映射 + 异步图库扩充 |

---

## 1. 产品背景与目标

### 1.1 产品定位

一款基于 AI 的**“一本正经胡说八道”**的情绪诊断工具。用户输入当前的负面情绪或状态，AI 将其映射为一种**“离谱的静物/动物”**（如：路边的减速带、公司的塑料绿植），并生成一张带有复古质感的“物种鉴定卡片”。

### 1.2 核心价值

* **用户侧：** 提供一种幽默的、低门槛的情绪宣泄出口和社交货币（标签化）。
* **产品侧：** 通过极具辨识度的视觉风格和扎心的文案，在小红书等社媒实现低成本裂变。

### 1.3 商业/成本策略

采用**“泛化映射 + 异步扩充”**策略。

* **绝大多数请求（99%）：** 使用 LLM 生成文案 + 调用预制图库（0.5秒响应，极低成本）。
* **极少数请求（1%）：** 触发“新物种发现”机制，实时生成新图并入库（高成本，用于制造惊喜和扩充图库）。

---

## 2. 核心用户流程 (User Flow)

1. **进入首页：** 看到“档案馆”氛围页，标本框内剪影闪烁。
2. **输入状态：** 用户在横线上输入文本（或点击“骰子”随机生成）。
3. **盖章提交：** 点击复古印章/开关，触发扫描动画。
4. **AI 分析与匹配：**
* LLM 分析情绪意象  匹配图库中最接近的 `Image_Tag`。
* LLM 生成“毒舌诊断”文案。


5. **结果展示：** 生成“黄金四段式”卡片。
6. **分享/保存：** 用户保存图片或分享链接。

## 3. 功能模块详解

### 3.1 首页：鉴定申请单 (Home)

**页面隐喻：** 一张放在旧木桌上的待填写的纸质申请单。

| 元素 | 需求描述 | 交互/逻辑 |
| --- | --- | --- |
| **Header** | 标题：**【 灵魂物种鉴定所 】**<br>

<br>Slogan：*Institute of Spiritual Speciation* | 静态展示，老报纸排版风格。 |
| **Visual** | **空置的标本框**（圆形或方形）。 | 循环播放离谱物种的**全黑剪影**，营造神秘感和正在扫描的氛围。 |
| **Input** | **状态主诉栏**。<br>

<br>样式：下划线填空式或老式打字机输入区。 | 限制 5-50 字。<br>

<br>字体：Typewriter (打字机字体)。 |
| **Tool** | **“懒人骰子”按钮**。 | 点击后随机填入预设的“发疯文案”（如：*“上班像上坟，心情如死灰”*）。 |
| **Trigger** | **【 盖章确认 】** 实体开关或红色火漆印章。 | 点击播放“咔嚓”盖章音效/震动。页面卷入/转场。 |

### 3.2 结果页：物种鉴定卡 (Result Card)

**页面隐喻：** 一张刚打印出来的、复古的博物馆藏品标签。

**卡片结构（黄金四段式）：**

| 区域 | 占比 | 内容与逻辑 |
| --- | --- | --- |
| **1. 视觉层** | 45% | **离谱物种大图**（复古铜版画/插画风格）。<br>

<br>来源：后端根据 Tag 匹配预制图库。 |
| **2. 身份层** | 10% | **物种名称**（超大字号，如【路边的减速带】）。<br>

<br>**稀有度**（右上角印章：SSR/SR/R，随机或根据逻辑生成）。 |
| **3. 标签层** | 15% | **3个核心形容词**（带#号，如 `#硬骨头 #专治不服`）。<br>

<br>作为视觉分割线。 |
| **4. 诊断层** | 30% | **扎心长文案**（30-50字，解释为何你是这个物体）。<br>

<br>**底部计数器**（见 3.3）。 |

### 3.3 计数器逻辑 (The Counter)

位于诊断文案的最下方，小字展示。

* **场景 A（匹配到存量图）：** 显示 `样本序列：No. [真实总生成数]`。
* *目的：* 制造“赛博病友”的共鸣感。


* **场景 B（触发新图生成）：** 显示 `样本序列：No. 0001 (新物种)` 且字体加红/加粗。
* *目的：* 给予“零号病人”的至高荣誉，刺激炫耀。

## 4. 后端与 AI 策略 (核心壁垒)

### 4.1 图库构建 (预制 + 异步)

* **冷启动库 (Seed Library)：** 预先生成 **50-80 张** 高质量复古风格图片。
* *覆盖分类：* 疲惫（烂泥）、愤怒（炸弹）、无用（空气）、坚硬（减速带）、伪装（塑料花）、阴暗（下水道）等。


* **匹配逻辑：**
* 用户输入  LLM 分析  输出 `Visual_Tag`。
* 如果 `Visual_Tag` 在库里（例如“疲惫”），随机调用一张“烂泥”或“咸鱼”的图。


* **异步扩充 (降级策略)：**
* 如果用户输入了极度冷门的意象（如“外星人”），且库里没有：
* **当前用户：** 强制映射到最接近的现有分类（如“异类”），用文案找补。
* **后台任务：** 记录“外星人”为高频缺失词，后台闲时生成新图入库。



### 4.2 Prompt 工程要求

LLM 需要输出严格的 JSON 格式，包含：

1. `object_name`: 物体名称（如：路边的减速带）
2. `visual_tag`: 视觉分类标签（用于后端调图）
3. `keywords`: 3个形容词标签
4. `diagnosis`: 毒舌诊断文案
5. `rarity`: 稀有度建议

## 5. UI/视觉设计规范

* **整体风格：** **复古百科全书 (Retro Encyclopedia) / 博物学风格**。
* **色板：**
* 背景：`#F0E6D2` (做旧羊皮纸色)
* 文字：`#2C2C2C` (墨色)
* 强调：`#B22222` (印章红)


* **字体：** 标题使用粗衬线体（老报宋），正文使用手写体或打字机体。
* **纹理：** 必须叠加噪点、纸张折痕、边缘磨损的 PNG 蒙版，避免数码味太重。

---

## 6. 数据埋点与运营指标

* **核心指标 (North Star Metric)：** **保存/截图率** (这是内容在小红书传播的前提)。
* **次级指标：**
* 首图生成时间 (目标 < 1.5s)。
* “懒人骰子”点击率 (衡量用户是否懒得输入)。
* 复玩率 (用户是否会测朋友/老板)。


* **内容监控：** 记录用户的高频输入词，用于迭代预制图库。
